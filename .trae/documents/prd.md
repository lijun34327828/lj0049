## 1. 产品概述

组件样式调试器（Component Style Debugger）是一款专注于前端 UI 组件样式实时调试的可视化工具。用户可在编辑界面中预览按钮、图标等组件，并通过右侧配置面板实时调整尺寸、配色、圆角、阴影等样式属性，支持默认、悬浮、点击三种状态独立配置，以及点击跳转、弹窗触发等交互行为设定，修改即时生效。

- 目标用户：前端开发者、UI 设计师
- 核心价值：零代码实时调试组件样式，告别反复修改 CSS 刷新页面的低效工作流

## 2. 核心功能

### 2.1 功能模块

1. **编辑界面（端口 3769）**：三栏布局，左侧组件选择面板、中间实时预览区、右侧样式配置面板
2. **预览界面（端口 8769）**：独立纯净预览窗口，仅展示当前配置的组件效果

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 编辑界面 | 组件选择面板 | 展示可选组件列表（按钮、图标），点击选中后加载到预览区 |
| 编辑界面 | 实时预览区 | 以交互式方式展示当前选中组件，支持悬浮/点击状态体验 |
| 编辑界面 | 样式配置面板 | 可视化配置组件的尺寸、配色、圆角、阴影等样式属性 |
| 编辑界面 | 状态切换器 | 切换默认/悬浮/点击三种状态的样式编辑 |
| 编辑界面 | 交互行为配置 | 配置点击跳转 URL、弹窗触发等交互行为 |
| 预览界面 | 纯净预览 | 独立窗口展示最终组件效果，无编辑器 UI 干扰 |

## 3. 核心流程

用户打开编辑界面 → 从左侧选择组件 → 中间预览区展示组件效果 → 右侧面板调整样式属性 → 选择不同状态（默认/悬浮/点击）分别配置 → 配置交互行为 → 所有修改实时同步到预览区 → 可在独立预览窗口（8769端口）查看纯净效果

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色编辑器风格（#1a1a2e 深蓝黑底），搭配 #00d4aa 青绿色高亮强调
- 次要色：#16213e 深蓝灰面板底色，#0f3460 中间层
- 按钮风格：圆角微透明，悬浮发光效果
- 字体：JetBrains Mono（代码/数值），Noto Sans SC（中文 UI 文案）
- 布局风格：三栏固定布局，左窄右宽，中间自适应
- 图标风格：线性图标，2px 描边，与整体暗色调一致

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 编辑界面 | 组件选择面板 | 深色侧栏、组件卡片列表、选中高亮边框 |
| 编辑界面 | 实时预览区 | 浅色/深色可切换背景、网格参考线、居中展示组件 |
| 编辑界面 | 样式配置面板 | 分组折叠面板、颜色选择器、滑块控件、数值输入框、状态标签切换 |
| 编辑界面 | 交互行为配置 | 下拉选择行为类型、URL 输入框、弹窗内容编辑区 |
| 预览界面 | 纯净预览 | 全屏居中展示、无编辑器 UI、组件交互效果完整体验 |

### 4.3 响应式设计

- 桌面优先设计，编辑界面最低支持 1280px 宽度
- 预览界面自适应窗口大小
- 右侧面板宽度固定 360px，左侧面板 240px，中间区域自适应
